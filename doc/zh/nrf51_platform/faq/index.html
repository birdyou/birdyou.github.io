故障排除

常见问题¶

1. 资源下载¶

所有的资源，包括开发环境、嵌入式代码、APP代码都开源了，请参见左侧使用说明。

2. 批处理烧写还是Keil烧写¶

默认配置下，Keil烧写应用层代码，nRFTools批处理脚本烧写的协议栈和SDK。
出厂的时候，板子默认已经烧写了协议栈和SDK，不用重复烧写。
所以，一般用户只需使用Keil烧写就可以了。

3. Keil无法识别出芯片¶

请确保在Keil中已经安装了nRF51的Package。
请确保使用JLink，并且配置为SWD模式。

4. 蓝牙搜不到¶

(1) Reset and Run
Keil中需要配置Jlink的reset and run，不然烧写代码后，需重新上电，程序才会运行。

(2) 供电正确
SMP支持CR2032纽扣电池供电。
JLink的供电也可以。
确保供电是3.3V。

(3) 手机开启/支持BLE
确保手机已经开启蓝牙和支持BLE。
Android 4.4+支持BLE，iPhone4以上才支持。
Android手机差异很大，建议多拿几个手机试试看。

(4) 手机使用了配套APP？
不要用手机原生的蓝牙搜索，BLE总是和配套APP使用的，手机原生的蓝牙搜索可能搜不到BLE设备。
iPhone用户可以安装一个lightblue的APP，它是通用的BLE调试工具。
当然，还是建议你安装我们的BLE调试助手APP来搜索和调试。

(5) Keil Debug
你也可以用Keil来debug，看看程序是否在运行？

(6) 物理损坏
还不行？看看板子是否有明显的物理损坏？

5. 蓝牙连不上¶

蓝牙可以搜到，但是连不上。
遇到这种情况，建议多换几个手机试试，然后确保板子供电3.3V。

6. nRFtools使用¶

nRFtools支持JLink，但不支持STLink，需要改驱动的。
